# Founder RC2 verification path

Do not mark `FOUNDER_EXPERIENCE_APPROVED` until you complete this path yourself.

## Setup

1. Use a private window.
2. Prefer a first-time account or clear first-use local storage so walkthrough v3 appears.
3. No founder coaching during the first pass.

## Walkthrough

1. Login.
2. Confirm the guide is visible (floating card).
3. Click the primary CTA (for example Open Needs me).
4. Confirm the guide **still visible** on the destination route.
5. Click Next. Confirm step advances.
6. Refresh. Confirm the same step returns.
7. Finish or Skip once. Confirm it does not auto-replay.

## Home

1. In five seconds, can you name what needs you?
2. Do you see Changed only when something real changed?
3. Is there a clear Next action or Talk invitation?
4. No S250 / SCALE_PROVEN / harness language.

## People

1. Open People.
2. Hover Shared projects or Recent collabs.
3. Confirm a preview or "No shared work yet."

## Invite (admin)

1. Open Users, Invite someone.
2. Confirm relationship (employee / contractor / …), team, manager, title.
3. Confirm no password field and plain language.

## Stop conditions (block RC2)

- Guide disappears when following a CTA.
- Home requires reading a paragraph of system status.
- Engineering proof text on Users or Home.
- Invite is only email + password-shaped create.

## After you approve

Tell the agent to set `FOUNDER_EXPERIENCE_APPROVED` and tag `OTZAR_YC_RC2`.
