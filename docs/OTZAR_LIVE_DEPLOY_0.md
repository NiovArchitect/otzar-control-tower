# OTZAR-LIVE-DEPLOY-0 — Deploy `api.otzar.ai` + `app.otzar.ai`

> Goal: Foundation API at **`https://api.otzar.ai`**, Otzar frontend at
> **`https://app.otzar.ai`**, HTTPS on, frontend → API wired, CORS allows the
> frontend, stored backend keys used. **The root `otzar.ai` and `www.otzar.ai`
> stay untouched** — we only add two new subdomain records.
>
> I have **no hosting-platform account/token**, so this is the exact runbook for
> you to execute. Where a value is generated by the platform (CNAME targets), I
> show exactly where to copy it from. **No secrets are in this file or in git.**

## What the repos support (Step 1 findings)
- **Foundation** (`niov-foundation`): production **Dockerfile** (Node 22, binds
  `0.0.0.0:3000`, `PORT` env), health route **`GET /api/v1/health`**. Needs:
  Postgres (**Supabase — already provisioned**), **Redis** (currently
  `localhost` → must use a managed Redis), and CORS via **`CONTROL_TOWER_URL`**.
  The repo's GitHub Actions deploy targets **AWS ECS / Azure** (heavy, gated on
  `CLOUD_TARGET`) — we will **not** use those; we deploy the Docker image directly
  on a simpler platform.
- **Otzar** (`otzar-control-tower`): plain **Vite static SPA** (Dockerfile builds
  → nginx; or any static host). API base = **`VITE_FOUNDATION_API_URL`**, used as
  the **full base including `/api/v1`** → value must be
  **`https://api.otzar.ai/api/v1`** (confirmed in `src/lib/api.ts`).

## 1. Recommended platform
**Render** — one platform for both, simplest for a DNS newcomer:
- Foundation → Render **Web Service** (from the repo Dockerfile).
- Redis → Render **Key Value** (managed Redis).
- Otzar → Render **Static Site** (Vite).
- Auto-HTTPS, custom domains via **CNAME**, per-service env + secrets.

Equally fine alternative: **Render for the API + Vercel (or Netlify) for the
frontend**. (Vercel/Netlify are excellent static hosts; see §3 note.)

## 2. Does the repo support it? — Yes
Foundation has a Docker web service; Otzar is a static Vite build. Both map
directly onto Render services. No code changes needed except the env/DNS wiring
below (plus the Dockerfile health-path fix already committed in
`[OTZAR-LIVE-DEPLOY-0]`).

## 3. Exact deployment steps (Render)

> Deploy the **API first** (you need its URL before building the frontend).

### A. Redis (Key Value)
1. Render dashboard → **New** → **Key Value** → name `otzar-redis` → create.
2. Copy its **Internal/External Redis URL** → this is `REDIS_URL`.

### B. Foundation API (Web Service)
1. **New** → **Web Service** → connect the `niov-foundation` repo (or "Deploy an
   existing image"). Environment: **Docker** (Render auto-detects the `Dockerfile`).
2. Name `otzar-api`. Region near your users. Instance: Starter (always-on; free
   tier sleeps on idle).
3. **Health Check Path:** `/api/v1/health`.
4. **Environment variables** (see §5 — set every one; mark keys as *secret*).
5. Create. Wait for build + a green health check. Note the generated URL
   `https://otzar-api.onrender.com`.
6. **Settings → Custom Domains → Add** `api.otzar.ai`. Render shows the **exact
   CNAME target** (e.g. `otzar-api.onrender.com`) — copy it for §4.

### C. Otzar frontend (Static Site)
1. **New** → **Static Site** → connect the `otzar-control-tower` repo.
2. **Build Command:** `npm ci && npm run build` · **Publish Directory:** `dist`.
3. **Environment variable:** `VITE_FOUNDATION_API_URL=https://api.otzar.ai/api/v1`.
4. Add a **rewrite** rule so the SPA routes resolve: source `/*` →
   destination `/index.html` (Action: Rewrite).
5. Create. Note the generated URL `https://otzar-app.onrender.com`.
6. **Custom Domains → Add** `app.otzar.ai`. Copy the **CNAME target** Render shows
   (e.g. `otzar-app.onrender.com`).

> After both custom domains are added in Render, do §4 (Namecheap), then return —
> Render issues HTTPS certs automatically once DNS resolves.

## 4. Exact Namecheap DNS records

> **Do NOT edit or delete the existing `@` (root) or `www` records.** Only **add**
> the two new records below. Both are **CNAME** (subdomains, not the apex).

| Type | Host | Value | TTL |
|---|---|---|---|
| CNAME Record | `api` | *(the CNAME target Render showed for `api.otzar.ai`, e.g. `otzar-api.onrender.com`)* | Automatic |
| CNAME Record | `app` | *(the CNAME target Render showed for `app.otzar.ai`, e.g. `otzar-app.onrender.com`)* | Automatic |

*If you use **Vercel** for the frontend instead:* add `app.otzar.ai` in Vercel →
it tells you to set **CNAME `app` → `cname.vercel-dns.com`** (use whatever value
Vercel displays). The `api` record stays pointed at Render.

### What to click in Namecheap
1. Log into **Namecheap** → **Domain List**.
2. **Manage** next to `otzar.ai`.
3. **Advanced DNS** tab.
4. Under **Host Records** → **Add New Record** (twice — one for `api`, one for
   `app`) with the values in the table. **Leave the existing records alone.**
5. **Save All Changes** (the green checkmark).
6. Wait for propagation (minutes–up to ~30; occasionally longer).
7. Verify:
   ```bash
   dig +short app.otzar.ai
   dig +short api.otzar.ai
   curl -I https://api.otzar.ai/api/v1/health      # expect HTTP/2 200 once live
   ```
